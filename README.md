# 食安多语风险图鉴 · FoodSafeML

> 通用 AI 智能体技能（Skill，跨平台）—— 给定任意食品名，产出可用于合规评估的 **多语 Excel《食品安全风险识别表》**，并留存标准原文与网页证据。**不依赖特定智能体平台**，可在 WorkBuddy / Claude Code / Cursor / OpenAI Codex / 通用 LLM+工具链 等任意支持 Skill 的环境运行。

- **技能 ID**：`foodsafeml`
- **版本**：`1.12.0`
- **适用**：任意支持「技能 / Skill」机制的 AI 智能体（运行平台无关；核心生成器为纯 Python）
- **变更日志**：见根目录 `CHANGELOG.md`

---

## 一、这个技能能做什么

输入一个食品名称（如「芒果干」「螺蛳粉」），技能会：

1. **引导录入**食品类别与配料（含添加剂 INS / E / CI / CAS 代码）；
2. **多语翻译** 14 种语言（简中 / 繁中 / 英 / 泰 / 高棉 / 越 / 阿 / 法 / 德 / 俄 / 西 / 葡 / 日 / 韩）；
3. **扩展别名检索**，并全球全量检索各国 / 地区 + ISO + WHO(经 JECFA) + FAO(Codex) + 各联盟 + 区域论坛的食品安全要求（执行标准 / 限量 / 检测方法 / 限制条件）；
4. **原文引用**，附来源网址 / 网页存档 / 发布日期 / 有效性；
5. 输出 **3 个工作表** 的多语 Excel。

## 二、工作簿结构（v1.12.0）

| 工作表 | 内容 |
|--------|------|
| **① 基础信息** | 食品基础字段列表 + 「配料多语对照」子表（14 语列） |
| **② 食品安全风险识别表** | 31 列源表：按全量指标清单（7 大类 126 项）遍历，有数据填值、无数据框架行标 `[待填写]`；含指标名称四语、法规名称三语、「声称执行标准」、限量值、检测方法、来源证据等 |
| **③ 指标对比查询** | 全量矩阵：126 指标 × 235 地区，按洲分区。有数据单元格 `HYPERLINK` 公式跳转源表或外链法规原文URL；采纳Codex/EU国家标注适用标准框架；不适用指标标"不适用"/"依产品标准评定"。绿/黄/红/白色标记。**来源可追溯** |

### 关键设计点

- **指标逐物质展开**：农残（毒死蜱、吡虫啉、氯氰菊酯…）、污染物（铅(Pb)、镉(Cd)、汞(Hg)、砷(As)…）逐项列出，不再使用笼统表述；
- **全量指标清单（7 大类 126 项）**：食品添加剂 18 + 农药残留 22 + 污染物 27 + 真菌毒素 12 + 微生物/致病菌 20 + **理化指标 17** + **感官指标 10**，覆盖理化、微生物、感官全维度，确保指标维度完整，不再因检索无数据而漏行；
- **全量地区矩阵**：**235 个**国家/地区+国际组织（全球主权国家+地区+ASEAN/GCC/EAEU/MERCOSUR/CARICOM/EAC/ECOWAS/SADC等联盟+CODEX/WHO/JECFA/FAO/ISO等国际组织），126×235 按洲分区矩阵；内置 Codex/EU/US 权威数据(BASE_LIMITS)+标准体系映射(采纳国标注适用标准框架+URL)，[待填写]占比约 0.7%（实测 0.69%，含用户 risks.json）；
- **强制微生物/致病菌类别**：菌落总数、大肠菌群、大肠杆菌、沙门氏菌、金黄色葡萄球菌、蜡样芽胞杆菌、霉菌、酵母、空肠弯曲菌、肉毒梭菌等（检索无覆盖时置 `[待填写]`）；
- **理化与感官维度**：理化指标（水分/酸价/过氧化值/挥发性盐基氮/组胺等）与感官指标（色泽/气味/滋味/组织状态/杂质/霉变/虫蛀等）全面纳入；感官指标统一标注「依产品标准评定」（非地区性数值限量）；
- **多语与合规**：指标名四语、法规名三语；新增「声称执行标准」字段（对应海外强制法规 / 自愿性认证标准，替代国内企业执行标准逻辑）；
- **防幻觉**：所有无法检索确认的多语字段 / 具体物质一律标注 `[待填写]`，绝不编造。

## 三、在各智能体平台中使用

本技能**不绑定任何特定平台**。核心资产是 `SKILL.md`（方法说明）+ `scripts/build_xlsx.py`（纯 Python 生成器，仅依赖 `openpyxl`）。把本仓库作为「技能 / Skill」加载到任意支持该机制的智能体即可，按需把文档中的「联网检索 / 提问 / 子智能体 / 写文件」替换为所在环境对应的能力。

### WorkBuddy
将本目录放进 `~/.workbuddy/skills/foodsafeml/`，然后在对话中说：
> 录入「芒果干」，出食品安全风险识别表，覆盖全球。
技能按 Phase 0–5 闭环执行（录入 → 翻译 → 扩展别名 → 检索归档 → 生成 Excel）。

### Claude Code / Codex / Cursor 等（以 Skills 目录加载）
将本仓库放到对应 skills 目录（如 `.claude/skills/foodsafeml/` 或项目内 `skills/foodsafeml/`），在对话中直接下达同样的指令即可；智能体会读取 `SKILL.md` 并调用 `scripts/build_xlsx.py`（需 `pip install openpyxl`）。

### 通用 LLM + 工具链（无内置 Skill 机制）
直接在提示词中粘贴 `SKILL.md` 的方法论，并让模型在有 Python 的环境执行 `python scripts/build_xlsx.py <食品目录> [食品名]`；所有「检索 / 提问 / 写文件」由你提供的工具完成。

> 注：无论哪种平台，Excel 生成这一步都依赖本地 Python + openpyxl；若运行环境无 Python，可改用等效的表格生成能力，但 `SKILL.md`「附录八」给出的单元格结构应保持一致以保证可追溯。

## 四、独立运行生成器（可选）

`scripts/build_xlsx.py` 是与 SKILL.md 附录八完全一致的独立可运行实现。给定某食品的
`basic.json` / `translations.json` / `risks.json`，可直接生成 Excel：

```bash
pip install openpyxl
python scripts/build_xlsx.py <食品目录> [<食品中文名>]
# 例：
python scripts/build_xlsx.py ./food-safety-芒果干 芒果干
```

生成产物：`<食品目录>/<食品名>_食品安全风险识别表.xlsx` 与 `standards/` 存档目录。
