# 食安多语风险图鉴 · FoodSafeML

> WorkBuddy 技能（Skill）—— 给定任意食品名，产出可用于合规评估的 **多语 Excel《食品安全风险识别表》**，并留存标准原文与网页证据。

- **技能 ID**：`foodsafeml`
- **版本**：`1.7.0`
- **适用**：WorkBuddy（`~/.workbuddy/skills/foodsafeml/`）

---

## 一、这个技能能做什么

输入一个食品名称（如「芒果干」「螺蛳粉」），技能会：

1. **引导录入**食品类别与配料（含添加剂 INS / E / CI / CAS 代码）；
2. **多语翻译** 14 种语言（简中 / 繁中 / 英 / 泰 / 高棉 / 越 / 阿 / 法 / 德 / 俄 / 西 / 葡 / 日 / 韩）；
3. **扩展别名检索**，并全球全量检索各国 / 地区 + ISO + WHO(经 JECFA) + FAO(Codex) + 各联盟 + 区域论坛的食品安全要求（执行标准 / 限量 / 检测方法 / 限制条件）；
4. **原文引用**，附来源网址 / 网页存档 / 发布日期 / 有效性；
5. 输出 **3 个工作表** 的多语 Excel。

## 二、工作簿结构（v1.7.0）

| 工作表 | 内容 |
|--------|------|
| **① 基础信息** | 食品基础字段列表 + 「配料多语对照」子表（14 语列） |
| **② 食品安全风险识别表** | 31 列源表：指标名称四语（简中/繁中/英/官方语言）、法规名称三语（简中全称/英全称/官方语言全称）、「声称执行标准」、限量值、检测方法、来源证据等 |
| **③ 指标对比查询** | 公式透视表：列头为「指标名称 \ 国家/地区(组织) *官方全称*」，所有数据单元格通过 `HYPERLINK` 公式从源表调取，**点击可直接跳转至源表对应原单元格** |

### 关键设计点

- **指标逐物质展开**：农残（毒死蜱、吡虫啉、氯氰菊酯…）、污染物（铅(Pb)、镉(Cd)、汞(Hg)、砷(As)…）逐项列出，不再使用笼统表述；
- **强制微生物/致病菌类别**：菌落总数、大肠菌群、大肠杆菌、沙门氏菌、金黄色葡萄球菌、蜡样芽胞杆菌、霉菌、酵母（检索无覆盖时置 `[待填写]`）；
- **多语与合规**：指标名四语、法规名三语；新增「声称执行标准」字段（对应海外强制法规 / 自愿性认证标准，替代国内企业执行标准逻辑）；
- **防幻觉**：所有无法检索确认的多语字段 / 具体物质一律标注 `[待填写]`，绝不编造。

## 三、在 WorkBuddy 中使用

将本目录放进 `~/.workbuddy/skills/foodsafeml/`，然后在对话中说：

> 录入「芒果干」，出食品安全风险识别表，覆盖全球。

技能会按 Phase 0–5 闭环执行（录入 → 翻译 → 扩展别名 → 检索归档 → 生成 Excel）。

---

## 四、独立运行生成器（可选）

`scripts/build_xlsx.py` 是与 SKILL.md 附录八完全一致的独立可运行实现。给定某食品的
`basic.json` / `translations.json` / `risks.json`，可直接生成 Excel：

```bash
pip install openpyxl
python scripts/build_xlsx.py <食品目录> [<食品中文名>]
# 例：
python scripts/build_xlsx.py ./food-safety-芒果干 芒果干
```

生成产物：`<食品目录>/<食品名>_食品安全风险识别表.xlsx` 与 `standards/` 存档目录。
